## ROLE

You are building **slabsvinyl.com** — a curated archive of new-condition, factory-sealed vinyl records. The aesthetic is archival, precise, and warm. Think industrial parts catalog meets record store liner notes. Every design decision should feel like it was made by a human who deeply understands both graphic design and vinyl culture — not generated by AI.

## BLOCK 2 — DESIGN SYSTEM DIRECTIVES

### Fonts — Load Method

**Instrument Serif** is loaded via `next/font/google` in `src/app/layout.tsx` and injected as a CSS variable (`--font-instrument-serif`).

**Funnel Display and Funnel Sans** are loaded via standard Google Fonts `<link>` tags in the `<head>` block of `src/app/layout.tsx`. Do NOT migrate them to `next/font/google` — they are intentionally kept as link tags.

The current working load pattern in `layout.tsx`:

```tsx
// next/font for Instrument Serif only
import { Instrument_Serif } from "next/font/google";

// Funnel Display + Funnel Sans via <head> link tags:
<head>
  <link rel="preconnect" href="https://fonts.googleapis.com" />
  <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="anonymous" />
  <link href="https://fonts.googleapis.com/css2?family=Funnel+Display:wght@300..800&family=Funnel+Sans:ital,wght@0,300..800;1,300..800&display=swap" rel="stylesheet" />
</head>
```

### Font Zone Rules — Non-Negotiable

| Font | Role | Size Range | Never use for |
|---|---|---|---|
| Instrument Serif | Headings, hero, editorial | 24px → 128px | Body copy, UI labels, metadata |
| Funnel Display | Nav, labels, overlines, archival metadata | 11px → 20px | Paragraph text |
| Funnel Sans | Body copy, product data, form fields | 12px → 18px | Headings above 18px |

- **Instrument Serif italic is the brand's emotional register** — use it for hero headlines and pull quotes only
- **Funnel Display at 11px + letter-spacing 0.08em + uppercase = the archival ZETR metadata style** — this is how all product spec data renders (artist, label, year, catalog number)
- **12px is the absolute floor.** No text ever renders below 12px.

### Color Rules — Strict

- **Amber (`#CD9121`) is BANNED as text on light backgrounds.** It fails WCAG AA (2.9:1 contrast). Use amber only on dark surfaces (`#141313`, `#242120`) or as a decorative 1px rule/dot.
- **Prices in light mode use Forest (`#204F41`).** Never amber in light mode.
- **Prices in dark mode use dark-accent (`#E8A830`).** Amber is valid here.
- The page background (`#F6F6F0`) is Paper — warmer than pure white, never use `#FFFFFF`.
- Surface (`#EFEEEA`) is Cream — used for cards and panels, slightly darker than Paper.
- Dark mode depth scale: `#141313` (base) → `#242120` (surface) → `#323030` (inputs/nested). These three steps are the only dark surfaces — do not invent new values.

### Dark Mode — How to Apply

Dark mode is toggled via `data-theme="dark"` on the `<html>` element. The `ThemeToggle` component (PR #21, pending merge) handles this. When applying dark mode styles:

- Use `dark:` Tailwind variants tied to the `dark-*` semantic tokens only — e.g. `dark:bg-dark-bg`, `dark:text-dark-text`
- Do not invent new dark surface colors — use only `dark-bg`, `dark-surface`, `dark-surface-muted`
- Apply `dark:` variants to every component that has a background, text color, or border
- Do not apply dark mode until PR #21 is merged to main. New components should include `dark:` class variants in anticipation, but they will have no effect until the toggle lands

### Border Radius — Zero Everywhere

```
border-radius: 0 on ALL components without exception.
```

The only exception in the entire codebase: the vinyl disc element uses `border-radius: 50%` because it represents a physical circle. Nothing else has any radius — not buttons, not inputs, not cards, not modals, not images, not badges.

### Border Style

All borders and dividers use alpha transparency — never solid gray:
- Borders (light): `rgba(20, 19, 19, 0.12)`
- Dividers (light): `rgba(20, 19, 19, 0.08)`
- Borders (dark): `rgba(239, 238, 234, 0.10)` — or token `border-dark-border`
- Dividers (dark): `rgba(239, 238, 234, 0.07)` — or token `border-dark-divider`

### Action Hierarchy — Four Levels Only

| Level | Visual | Use |
|---|---|---|
| Primary | Forest bg (`#204F41`) + Cream text (`#EFEEEA`), full width, 14px uppercase | "Grab it" — PDP only |
| Secondary / Ghost | 1px border (`rgba(20,19,19,0.12)`), no fill, text-muted color | "Add to wishlist", secondary PDP actions |
| Tertiary | Text link + `→` suffix, no border, no bg | "View all →", collection CTAs |
| Quaternary | Underline only, `text-underline-offset: 2px` | Inline editorial links, sonic lineage artist names |

No other button or link styles exist. No colored backgrounds on secondary actions.

### Motion — Analog Feel

All transitions use `cubic-bezier(0.25, 0.1, 0.25, 1)` — smooth and deliberate, never bouncy or springy. Duration reference:
- `120ms` — micro interactions (button press, color shift)
- `180ms` — hover state transitions
- `320ms` — drawer open/close, standard UI
- `300ms` — vinyl disc slide on product card hover
- `600ms` — crate dig full-card transitions

### Status Labels

Status words ("New", "Gone", "Last Copy") are **plain text only**. No pill, no background, no border, no border-radius. Font: Funnel Display, 10px, weight 600, letter-spacing 0.10em, uppercase.
- New → color `#4E4843` (text-muted)
- Gone → color `#9A181A` (wax/error)
- Last Copy → color `#B28B58` (caramel)

### Icons

Use Lucide icons only. Always 16px (UI) or 20px (nav). Stroke weight: 1.5px. Color: always text-muted (`#4E4843` light / `#A09589` dark). Never filled. Never colored. Add `aria-label` to all icon-only buttons.

### Layout & Grid

- Max content width: `1280px`, centered, side gutters: `clamp(1.5rem, 5vw, 5rem)`
- Product grid: 4 columns desktop / 2 columns tablet (≤1024px) / 1 column mobile (≤640px)
- Product grid gap: `1rem` (16px)
- All section vertical padding: `clamp(4rem, 8vw, 8rem)`
- Body text max-width: `65ch`

### Homepage Section Order — Canonical

The `src/app/page.tsx` file must render sections in exactly this order:

```
1. Nav
2. Hero
3. NewArrivals
4. FeaturedCollections
5. FromTheCrates       ← not yet built
6. SonicLineage        ← in progress
7. AboutTrust
8. Footer
```

Footer is also rendered globally in `src/app/layout.tsx`. Do not duplicate it. When adding a new section to `page.tsx`, place it in the position above — do not append to the bottom without checking order first.

## BLOCK 3 — HARD CONSTRAINTS

These are explicit rejections. If the builder would normally do any of these, do not do them here.

### Never Do
- `border-radius` on any element except the vinyl disc (`border-radius: 50%`)
- Gradient backgrounds or gradient buttons — use solid fills only
- Amber (`#CD9121`) as text on any light surface
- Purple, blue, or teal as primary colors — the brand primary is Forest green (`#204F41`)
- Pill-shaped badges or chips — status is text only
- Buy buttons, "Add to Cart" badges, or any interactive overlay on album cover images — the image is pure, untouched
- Colored side borders on cards (`border-left: 3px solid ...`) — cards use neutral alpha borders
- Icons inside colored circles or rounded squares
- Centered body text — left-align all body copy, product metadata, and card content
- `font-family: Inter`, `Roboto`, `system-ui` or any font not in the approved set
- White (`#FFFFFF`) as a background — use Paper (`#F6F6F0`) or Cream (`#EFEEEA`)
- Shadows with pure black (`rgba(0,0,0,...)`) — all shadows use warm tones
- Any font size below 12px
- Tailwind default color names (`blue-500`, `gray-200`, etc.) — use only the Slabs color tokens defined in the config above
- Default Tailwind border-radius classes (`rounded`, `rounded-lg`, etc.) — they are all overridden to `0`
- Migrating Funnel Display or Funnel Sans to `next/font/google` — they must stay as `<link>` tags

### Product Card Specific
- No buy button in the product grid — "Grab it" lives on the product detail page only
- The album cover image links to the PDP — it is the only interactive element on the card
- Vinyl hover: the disc slides OUT from behind the sleeve to the right (`translateX(55%)`) — not inward, not upward
- Container of the vinyl animation must use `overflow: visible` — otherwise the disc disappears into the sleeve

### Typography Specific
- Never use Instrument Serif below 24px
- Never use Funnel Display for paragraph body copy
- Never mix Instrument Serif and Funnel Display at the same rendered pixel size on screen
- Funnel Display metadata (artist, label, year, catalog numbers) must always be uppercase with letter-spacing ≥ 0.04em
